# principle.kb/ — durable rules of thumb for future work

## What belongs

Generalizable lessons of the form "always do X" or "watch out for Y."
Distilled from one or more concrete observations / failures / corrections.
Each file states the principle, sketches the failure mode it guards
against, and points at the originating evidence.

Also: durable user policies — they share the "always-applies" shape with
methodological principles.

## What does NOT belong

- **Specific decisions** for one fork → `../decision.kb/`.
- **Observations** about current state → `../observation.kb/`.
- **One-off corrections** not yet generalizable into a principle —
  leave in observation.kb/ until a second instance triggers promotion.

## When to add

When a pattern recurs across two or more instances, or when a single
instance is striking enough that you want to bias future work
permanently. Don't add a principle for a one-off that might not
generalize; you'll just create stale advice.

# Session-start ritual

## Steps

1. Read `../../rust-port.md` — particularly `## Working stance` (re-anchor if it's been a while) plus any TOC changes.
2. `Bash ls ../handoffs.kb/` — scan pending topics. Read those that may apply; when acting on one, follow `consume-handoff.md`.
3. Read the **previous** commit doc (next lower ordinal in `commits.kb/`) — focus on its `## Outcomes` and `## Notes` for resume context.
4. On-demand from the current commit's `## Refs`: contracts, facts, decisions.

Ancestor `CLAUDE.md` files (user-global, project, package, kb chain) auto-load and provide section semantics and kb boundaries. No need to re-read them unless something specifically calls for it.

## Throughout the session

- Watch for hand-off-write opportunities — mid-work discoveries affecting *future* commits. Triggers and format: `../handoffs.kb/CLAUDE.md`.
- When acting on a hand-off, run promote-vs-delete via `consume-handoff.md`.

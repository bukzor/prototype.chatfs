# Technical Policy

Cross-cutting normative guidance — how we achieve requirements (means, not ends).

## What belongs here

- Design rules and constraints that apply across components
- Convention decisions with rationale
- Policies that prevent re-litigating settled questions

## What does NOT belong

- Component-specific design (→ design-incubators or technical-design)
- Goals or requirements (→ design knowledge stack)
- Implementation details (→ code)
